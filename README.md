# Фронтенд
### **Файл index.html: разметка страницы**

- В верхней части расположена шапка с логотипом, поисковой строкой и кнопкой авторизации (демо-режим). Ниже находится панель навигации с шестью разделами: Главная, Галерея, Хронология, Архив, Интерактив и Блог. Переключение между разделами происходит без перезагрузки страницы;
- Раздел «Главная страница» включает приветственный баннер с кнопками перехода в галерею и выбора случайного мема, блок «Популярные мемы недели» с четырьмя карточками и анонс новой эпохи мемов 2024 года;
- Раздел «Галерея» содержит сетку карточек всех мемов и панель фильтрации по трем эпохам: 2000-е (классика), 2010-е (виральность) и 2020-е (TikTok эра);
- Раздел «Хронология» представляет три временных периода с карточками мемов внутри каждой эпохи и текстовым блоком с историческим контекстом;
- Раздел «Архив» включает облако тегов, блок редких мемов и форму для предложения нового мема;
- Раздел «Интерактив» содержит три элемента: квиз «Угадай мем» с несколькими вопросами, генератор мемов (пользователь вводит верхний и нижний текст, получает превью) и голосование за лучший котомем 2024 года;
- Раздел «Блог» включает видеолекции и статьи о мемах, а также глоссарий терминов;
- В конце страницы расположено модальное окно для детального просмотра мема: при клике на любую карточку открывается окно с полной историей, тегами, изображением и списком похожих мемов, по которым также можно кликнуть. Завершает страницу подвал с копирайтом и иконками социальных сетей;

### **Файл style.css: визуальное оформление**

- Цветовая схема выполнена в теплых нейтральных тонах: основной фон #fbfaf7, акцентный цвет #cf6f2e, темный текст #1e1b1a. Для типографики используются шрифты Inter (основной текст) и Space Grotesk (логотип);
- Карточки мемов имеют скругленные углы (24px), легкую тень и плавную анимацию при наведении: карточка приподнимается, а изображение внутри масштабируется. Кнопки и фильтры выполнены со скруглением 40px, при наведении меняют цвет с нейтрального на акцентный;
- Модальное окно реализовано с затемненным фоном, центрированным содержимым и возможностью прокрутки при длинном тексте. Закрытие происходит по клику на крестик или на фон вокруг окна;
- Реализована адаптивная верстка с медиа-запросом для устройств с шириной экрана до 700 пикселей: навигация перестраивается в колонку, уменьшаются отступы и размеры шрифтов. Добавлен кастомный скроллбар в оранжевых тонах;

### **Файл script.js: логика и данные**

Файл script.js содержит всю логику сайта и базу данных из 31 мема. Каждый мем включает: id, название, эпоху, теги, историю, похожие мемы, изображения.

*Распределение по эпохам:*

- 2000-е: Lolcat, Longcat, Ceiling Cat, «Погладь кота» и др.;
- 2010-е: Grumpy Cat, Кричащий кот, Кот пилит ногти, Thurston, Максвелл;
- 2020-е: 20 мемов: Мистер Фреш, Crunchy cat, Vibing cat, Уставший кот, Sad banana cat, Dancing cat, Кот за рулем, Шлепа (каракал Гоша) и др.

*Основные функции:* показ деталей мема, закрытие окна, галерея с фильтром по эпохе, хронология, облако тегов, глоссарий, квиз (9 вопросов), голосование, переключение разделов.
*Интерактив:* поиск по названию, фильтрация, генератор мемов с текстом, голосование за лучший мем, переходы по похожим мемам.
*Навигация:* 6 разделов, переключение без перезагрузки, логотип ведет на главную.
*Внешние ссылки:* Видеолекция на YouTube про мем «А?» и статьи о мемах.
*Технические особенности проекта:* Код организован модульно: HTML, CSS и JavaScript разделены по отдельным файлам, что упрощает поддержку и расширение проекта. Все данные о мемах хранятся локально в массиве memesData, что делает сайт полностью автономным - не требуется подключение к внешнему API или серверу для отображения контента.

### **Файлы auth: авторизация**

Была добавлена полноценная система входа и регистрации.

*auth-component.html* - модальное окно для входа и регистрации. В нем:

- Две вкладки - «Войти» и «Регистрация»;
- Поля для ввода логина, пароля, email (при регистрации);
- Кнопка закрытия.

*auth-styles.css* - стили для этого окна. В нем: плавное появление, скругленные углы, приятные цвета. Окно затемняет фон страницы и открывается по центру.

*auth-script.js* - главная логика авторизации.

- Хранит данные пользователя в localStorage (чтобы не входить каждый раз заново);
- Отправляет запросы на сервер для входа и регистрации;
- Показывает имя пользователя в шапке вместо кнопки «Войти», когда кто-то зашел;
- Дает кнопку «Выйти», чтобы выйти из аккаунта.

*Как это работает вместе:* пользователь нажимает на кнопку «Войти» в шапке - открывается окно. Вводит логин и пароль - отправляется запрос на сервер. Если все правильно, в шапке появляется его имя и кнопка «Выйти». Если пользователь закроет и откроет сайт снова - он останется в системе.

# Бэкенд

Бэкенд представляет собой REST API, реализованный на фреймворке FastAPI. Серверная часть обеспечивает хранение данных в PostgreSQL, кэширование через Redis, полнотекстовый поиск через Elasticsearch, а также авторизацию пользователей с JWT-токенами. Вся инфраструктура контейнеризирована с помощью Docker и управляется через docker-compose.

### Модели базы данных (app/models)

**user.py - пользователи и авторизация**

User хранит учетные записи: имя пользователя, электронную почту, hashed_password, роль, аватары, биографию, счетчики вкладов и голосов. Роли определяют права доступа к редактированию контента. MemeSuggestion служит для сбора предложений новых мемов от пользователей.

**meme.py - главная сущность мема**

Meme содержит заголовок, URL-идентификатор, ссылки на изображение и видео, описание, историю происхождения, культурный контекст, год создания, связь с эпохой, источник, оригинальную платформу, счетчики просмотров и лайков, а также флаги is_rare и is_nsfw. Таблица meme_tag_association связывает мемы с тегами (многие-ко-многим). Таблица similar_memes задает связи между похожими мемами с оценкой схожести.

**era.py - исторические эпохи**

Era описывает временные периоды: название, начальный и конечный год, описание, культурный контекст, ключевые события, обложку, цветовую метку. EraMapPoint добавляет координаты, заголовок и описание для каждой эпохи.

**article.py - статьи и видеолекции**

Article предназначена для блога: заголовок, тип (статья/лекция), контент, краткое описание, обложка, ссылка на видео, связь с автором, эпохой и анализируемым мемом, статус (черновик/опубликовано), дата публикации, счетчик просмотров, время чтения и мета-данные. Таблица article_tags хранит теги для статей.

**source.py - первоисточники**

Source фиксирует платформы и сайты, где впервые появился мем: название, тип (платформа/сайт), описание, ссылка, связь с эпохой.

**tag.py - теги**

Tag содержит теги для мемов и статей: название, slug, категорию, цвет, счетчик использований, флаг is_nsfw.

### API-эндпоинты (app/routers)

**auth.py - авторизация**

Эндпоинты для регистрации (POST /auth/register), входа (POST /auth/login) и получения текущего пользователя (GET /auth/me). Используется JWT с bcrypt-хешированием паролей. Токен возвращается в ответе при успешном входе/регистрации.

**gallery.py - галерея мемов**

Основной эндпоинт GET /gallery/memes поддерживает пагинацию, фильтрацию по эпохе, тегу, году, формату, поиск по тексту, сортировку (по дате, году, просмотрам, лайкам). Эндпоинт GET /gallery/memes/{slug} возвращает детальную информацию о конкретном меме и автоматически увеличивает счетчик просмотров. GET /gallery/memes/{id}/similar возвращает похожие мемы на основе общих тегов. GET /gallery/filter/eras и /gallery/filter/tags отдают данные для фильтров на фронтенде. Эндпоинт создания мема (POST /gallery/memes) доступен только редакторам.

**chronology.py - хронология**

GET /chronology/eras возвращает список всех эпох с количеством мемов в каждой. GET /chronology/eras/{slug} отдает детали конкретной эпохи с карточками мемов и точками на карте. GET /chronology/timeline формирует таймлайн с группировкой мемов по годам и тремя самыми популярными мемами на каждый год. GET /chronology/map/{era_slug} возвращает географические точки для карты эпохи. GET /chronology/context/{era_slug} возвращает культурный контекст и ключевые события.

**search.py - поиск**

Глобальный поиск GET /search/ ищет по всем разделам: мемам, статьям, тегам, эпохам. Поддерживается фильтрация по типу контента. Эндпоинт автокомплита GET /search/autocomplete возвращает подсказки для поисковой строки (мемы, теги, эпохи) по мере ввода текста.

### Инфраструктура и развертывание

**Dockerfile**

Устанавливает системные зависимости (gcc, libpq-dev), копирует requirements.txt и устанавливает пакеты, копирует исходный код и статику, создает папку для загрузок, запускает uvicorn с 4 воркерами на порту 8000.

**docker-compose.yml**

Описывает 4 сервиса:

- api - сам бэкенд, собирается из Dockerfile, монтирует папку uploads, зависит от БД, Redis и Elasticsearch.
- postgres - база данных PostgreSQL 15, хранит данные в томе.
- redis - кэш-сервер Redis 7.
- elasticsearch - поисковый движок Elasticsearch 8.11.0 для полнотекстового поиска.

**main.py**

Точка входа FastAPI: настройка CORS (разрешены запросы с localhost:3000 и домена музея), монтирование статических файлов из папки frontend и загруженных медиа, подключение роутеров (auth, gallery, chronology, search), эндпоинт /ui для отдачи фронтенда, эндпоинт /health для проверки работоспособности. Lifespan управляет созданием папок и закрытием соединений при остановке.

**config.py**

Управляет настройками через переменные окружения: секретный ключ для JWT, параметры алгоритма, срок жизни токена, строка подключения к PostgreSQL, URL Redis и Elasticsearch, путь для загрузки файлов.

**requirements.txt**

Список зависимостей: FastAPI, Uvicorn, SQLAlchemy с asyncio, asyncpg, Pydantic, python-jose (JWT), passlib + bcrypt, redis (асинхронный), elasticsearch (асинхронный), python-multipart, aiofiles.
